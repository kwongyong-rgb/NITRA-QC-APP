NITRA QC App v8 - PDF report and inspection rule update

Implemented changes:
1. Renamed PDF report explanation section to Inspection Evaluation Criteria.
2. Added concise Visual and Technical sampling/fail criteria:
   - Visual: <=100 pcs inspect 8, each additional 100 pcs +4.
   - Visual fail follow-up: 1 initial fail -> +4 for same defect; same defect again -> 100%; 2+ initial fails -> immediate 100%.
   - Technical: <=100 pcs inspect 4, each additional 100 pcs +2.
   - Technical fail follow-up: 1 initial fail -> +2 for same defect; same defect again -> 100%; 2+ initial fails -> immediate 100%.
3. Consolidated Inspection Outcome and 100% Inspection Results into one table showing every parameter with checked/pass/fail/failing pieces/outcome.
4. Defect Log now sorts by inspected parameter first, then piece number.
5. Defect Log removed Type, Severity, Value and Stage columns.
6. Defect Log photo icon now links to the actual signed photo/video URL when available.
7. Photo Appendix now labels pieces as Piece 3 instead of pc3.
8. Photo Appendix images/videos are clickable and open the full-size image/video URL.
9. Updated rule engine to use Visual extra follow-up = 4 and Technical extra follow-up = 2.
10. Updated rule engine to trigger immediate 100% inspection when 2+ initial sample pieces fail for the same defect/parameter.
11. Updated Visual tab logic so all Visual parameters use the Visual sample size.

Validation:
- npm install --ignore-scripts completed.
- npm run build completed successfully.
